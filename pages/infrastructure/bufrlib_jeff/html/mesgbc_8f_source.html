11 C> - If LUNIN > 0, the subroutine reads and examines Section 1 of each

13 C> continuing until it reaches the first message which actually contains
14 C> report data. This means it will skip any messages containing DX BUFR
15 C> table information, as well as any "dummy" messages containing the
16 C> dump center time or dump initiation time within NCEP dump files.
17 C> It then returns the message type and compression indicator from the
18 C> first message containing report data. When used this way, the
19 C> BUFR file in question should not have already been opened via a call
20 C> to subroutine openbf(), though it should already be associated with
21 C> Fortran logical unit number LUNIN. In this situation, the
22 C> subroutine is similar to subroutine mesgbf(), except that mesgbf()
23 C> doesn't skip past any "dummy" messages within NCEP dump files, nor
24 C> does it return a compression indicator.
25 C> - If LUNIN < 0, the subroutine simply returns the message type and
26 C> compression indicator for the BUFR message currently stored in the
